# CLAUDE.md

## Project Overview

Tree-sitter grammar for the Crystal programming language. Defines the full Crystal syntax in `grammar.js` with an external C scanner for strings, heredocs, regex, and other context-sensitive tokens.

## Commands

```bash
npm run generate          # Generate parser from grammar.js (outputs src/parser.c)
npm run build             # Generate + compile native addon
npm run test              # Run the test corpus
npm run parse -- file.cr  # Parse a single file (debugging)
```

## Architecture

### Key Files

| File | Purpose |
|------|---------|
| `grammar.js` | Main grammar definition — single source of truth for all Crystal syntax rules |
| `src/scanner.c` | External scanner (C) for context-sensitive tokens: strings, interpolation, heredocs, regex, command literals |
| `src/parser.c` | Auto-generated by `tree-sitter generate` — do not edit |
| `src/grammar.json` | Auto-generated metadata — do not edit |
| `tree-sitter.json` | Tree-sitter project config (scope, file-types, bindings) |

### Queries

- `queries/highlights.scm` — Syntax highlighting
- `queries/tags.scm` — Tags/symbols for navigation
- `queries/locals.scm` — Local variable scoping

### Test Corpus

Tests in `test/corpus/*.txt` using tree-sitter's standard corpus format (input source paired with expected S-expression trees).

### Examples

`examples/` — Example Crystal files for manual parse testing.

## Dependencies

| Package | Purpose |
|---------|---------|
| `tree-sitter-cli` ^0.26.6 | CLI for generating and testing the parser |
| `node-addon-api` ^8.3.1 | Node.js native addon API |
| `node-gyp-build` ^4.8.4 | Build tool for native addons |

## Notes

- Targets Crystal `.cr` files (scope: `source.crystal`)
- Bindings configured for C, Node, Rust, Go, Python, and Swift
- After any change to `grammar.js` or `src/scanner.c`, run `npm run generate` then `npm test`
- Never edit `src/parser.c` or `src/grammar.json` by hand
